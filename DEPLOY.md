# Deployment Guide — Render (Single Service)

Everything — the React frontend, Express API, and background scheduler — runs as **one web service** on Render, backed by a Render PostgreSQL database. No Vercel needed.

```
Render Web Service  (Express serves both /api routes and the React frontend)
Render PostgreSQL   (free, auto-connected via DATABASE_URL)
```

---

## Step 1 — Push to GitHub

1. Go to [github.com/new](https://github.com/new) → create a **new empty repo** (no README, no .gitignore)

2. In the Replit Shell, run:
   ```bash
   git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
   git push -u origin main
   ```

Push the **entire monorepo root** — not a subfolder.

---

## Step 2 — Deploy with the Render Blueprint (one click)

The repo already includes `render.yaml` which defines the web service + database.

1. Go to [dashboard.render.com/blueprints](https://dashboard.render.com/blueprints)
2. Click **New Blueprint Instance**
3. Connect your GitHub account and select the repo
4. Render detects `render.yaml` automatically
5. Fill in the three secret values it asks for (see below)
6. Click **Apply** — Render creates both the database and web service together

---

## Step 3 — Set environment variables

When the Blueprint prompts for values, enter:

| Variable | What to enter |
|----------|---------------|
| `DASHBOARD_PASSWORD` | Your chosen login password (e.g. `mysecretpass123`) |
| `X_CLIENT_ID` | From the X Developer Portal (see Step 4) |
| `X_CLIENT_SECRET` | From the X Developer Portal (see Step 4) |

`SESSION_SECRET` is auto-generated by Render — you don't need to set it.  
`DATABASE_URL` is auto-connected from the Render PostgreSQL database.

---

## Step 4 — Set up your X Developer App

You need an X (Twitter) Developer account and app to get the OAuth credentials.

1. Go to [developer.twitter.com](https://developer.twitter.com) → sign in
2. Create a new project + app (or use an existing one)
3. In your app settings, go to **User authentication settings**:
   - **App permissions**: Read and Write
   - **Type of App**: Web App
   - **Callback URI**: *(leave blank for now — you'll fill this in after deploy)*
4. Go to **Keys and tokens** → copy **Client ID** and **Client Secret** (OAuth 2.0)
5. Paste those into the Render env vars from Step 3

---

## Step 5 — Run the database migration

After the first successful deploy (green checkmark in Render):

1. In Render, open your web service → **Shell** tab
2. Run:
   ```bash
   pnpm --filter @workspace/db run push
   ```
   This creates all the tables (campaigns, urls, posts, x_account).

---

## Step 6 — Add the callback URL to your X app

Once deployed, Render gives your service a URL like:
```
https://auto-x-poster.onrender.com
```

1. Go back to your X Developer app → **User authentication settings**
2. Set **Callback URI / Redirect URL** to:
   ```
   https://auto-x-poster.onrender.com/api/x-account/callback
   ```
3. Save

---

## Step 7 — First login and connect X

1. Open your Render URL in a browser (e.g. `https://auto-x-poster.onrender.com`)
2. Enter your `DASHBOARD_PASSWORD`
3. Go to **Settings → Connect X Account**
4. Complete the Twitter OAuth flow — you'll be redirected back to Settings showing "CONNECTED"
5. Go to your campaign → add some URLs → click **Force Post** to test

## Free plan notes

| Thing | Detail |
|-------|--------|
| **Sleep** | Free services sleep after 15 min of no web traffic |
| **Wake-up** | When you open the app, it wakes up in ~30 seconds |
| **Scheduler** | Runs every 5 min when awake; catches up on due posts when it wakes |
| **1-hour post frequency** | Fully supported — if the service sleeps between posts, it catches up next visit |
| **DB** | Render PostgreSQL free tier expires after 90 days — upgrade or export before then |
| **Keep-alive (optional)** | Use [UptimeRobot](https://uptimerobot.com) free plan to ping `/api/healthz` every 5 min — keeps the service awake 24/7 |

## Manual deploy settings (if you don't use the Blueprint)

If you prefer to create the service manually instead of using the Blueprint:

**Web Service:**
| Setting | Value |
|---------|-------|
| Root Directory | *(leave blank — repo root)* |
| Build Command | `chmod +x ./render-build.sh && ./render-build.sh` |
| Start Command | `node --enable-source-maps artifacts/api-server/dist/index.mjs` |
| Health Check Path | `/api/healthz` |

**Environment Variables** (add manually):
- `NODE_ENV` = `production`
- `DATABASE_URL` = *(from Render PostgreSQL internal URL)*
- `SESSION_SECRET` = *(any long random string)*
- `DASHBOARD_PASSWORD` = *(your password)*
- `X_CLIENT_ID` = *(from X Developer Portal)*
- `X_CLIENT_SECRET` = *(from X Developer Portal)*
