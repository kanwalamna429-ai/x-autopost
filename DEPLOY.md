# Deployment Guide — Render (Single Service)

Everything — the React frontend, Express API, and background scheduler — runs as **one web service** on Render, backed by a Render PostgreSQL database.

```
Render Web Service  →  Express server (API + scheduler + serves the React UI)
Render PostgreSQL   →  Free, auto-connected via DATABASE_URL
UptimeRobot         →  Free pinger, keeps Render awake 24/7 so scheduler never stops
```

> **Important:** Render's free plan sleeps after 15 min of no traffic, which kills the scheduler.
> You **must** complete Step 8 (UptimeRobot) for fully automatic posting without needing to open the app.

---

## Step 1 — Push to GitHub

1. Go to [github.com/new](https://github.com/new) → create a **new empty repo** (no README, no .gitignore)

2. In the Replit Shell, run:
   ```bash
   git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
   git push -u origin main
   ```

Push the **entire monorepo root** — not a subfolder.

---

## Step 2 — Deploy with the Render Blueprint (one click)

The repo includes `render.yaml` which defines the web service + database together.

1. Go to [dashboard.render.com/blueprints](https://dashboard.render.com/blueprints)
2. Click **New Blueprint Instance**
3. Connect your GitHub account and select the repo
4. Render detects `render.yaml` automatically
5. Fill in the three secret values it asks for (see Step 3)
6. Click **Apply** — Render creates the database and web service automatically

---

## Step 3 — Set environment variables

When the Blueprint prompts for values, enter:

| Variable | What to enter |
|----------|---------------|
| `DASHBOARD_PASSWORD` | Your chosen login password (e.g. `mysecretpass123`) |
| `X_CLIENT_ID` | From the X Developer Portal (see Step 4) |
| `X_CLIENT_SECRET` | From the X Developer Portal (see Step 4) |

`SESSION_SECRET` is auto-generated by Render.
`DATABASE_URL` is auto-connected from the Render PostgreSQL database.

---

## Step 4 — Set up your X Developer App

1. Go to [developer.twitter.com](https://developer.twitter.com) → sign in
2. Create a new project + app (or use existing)
3. In **User authentication settings**:
   - App permissions: **Read and Write**
   - Type of App: **Web App**
   - Callback URI: *(leave blank for now — fill in after deploy)*
4. Go to **Keys and tokens** → copy **Client ID** and **Client Secret** (OAuth 2.0)
5. Paste those into the Render env vars from Step 3

---

## Step 5 — Run the database migration

After the first successful deploy (green checkmark in Render):

1. Open your web service in Render → **Shell** tab
2. Run:
   ```bash
   pnpm --filter @workspace/db run push
   ```
   This creates all tables (campaigns, urls, posts, x_account).

---

## Step 6 — Add the callback URL to your X app

Once Render finishes deploying, your app URL looks like:
```
https://auto-x-poster.onrender.com
```

1. Go to your X Developer app → **User authentication settings**
2. Set **Callback URI** to:
   ```
   https://auto-x-poster.onrender.com/api/x-account/callback
   ```
3. Save

---

## Step 7 — Connect your X account

1. Open your Render URL in a browser
2. Log in with your `DASHBOARD_PASSWORD`
3. Go to **Settings → Connect X Account**
4. Complete the Twitter OAuth flow
5. Go to a campaign → add URLs → click **Force Post** to test

---

## Step 8 — Set up UptimeRobot (REQUIRED for autopilot)

Without this step, Render sleeps the server after 15 minutes and the scheduler stops.
UptimeRobot is free and pings your app every 5 minutes to keep it awake 24/7.

1. Sign up at [uptimerobot.com](https://uptimerobot.com) — free plan is enough
2. Click **Add New Monitor**
3. Fill in:

   | Field | Value |
   |-------|-------|
   | Monitor Type | **HTTP(s)** |
   | Friendly Name | Auto X Poster |
   | URL | `https://auto-x-poster.onrender.com/api/healthz` |
   | Monitoring Interval | **5 minutes** |

4. Click **Create Monitor**

That's it. UptimeRobot will ping your app every 5 minutes. Render never sleeps.
The scheduler checks for due posts every 5 minutes and posts automatically on schedule.

---

## How autopilot works after setup

```
UptimeRobot pings /api/healthz  →  every 5 minutes (keeps Render awake)
Scheduler checks campaigns      →  every 5 minutes (same loop)
Campaign is due?                →  posts tweet to X, updates nextPostAt
Next post scheduled             →  now + frequency (e.g. 1 hour later)
```

You never need to open the app. Just set up campaigns, add URLs, and let it run.

## Manual deploy settings (if not using the Blueprint)

**Web Service settings:**

| Setting | Value |
|---------|-------|
| Root Directory | *(leave blank)* |
| Build Command | `chmod +x ./render-build.sh && ./render-build.sh` |
| Start Command | `node --enable-source-maps artifacts/api-server/dist/index.mjs` |
| Health Check Path | `/api/healthz` |

**Environment Variables:**
- `NODE_ENV` = `production`
- `DATABASE_URL` = *(Render PostgreSQL internal URL)*
- `SESSION_SECRET` = *(any long random string)*
- `DASHBOARD_PASSWORD` = *(your password)*
- `X_CLIENT_ID` + `X_CLIENT_SECRET` = *(from X Developer Portal)*
